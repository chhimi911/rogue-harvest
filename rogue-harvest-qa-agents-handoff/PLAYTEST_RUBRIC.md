# PLAYTEST_RUBRIC.md

## Purpose

Use this rubric to judge Rogue Harvest's current playable quality.

Scores are 1–5.

- 1 = broken or confusing
- 2 = works poorly
- 3 = acceptable MVP
- 4 = strong demo
- 5 = polished public demo

## Rubric

| Area | Score | What to Check |
|---|---:|---|
| Core loop works |  | Start run → battle → reward → next node → win/loss |
| Combat clarity |  | HP, block, energy, hand, intent, damage feedback |
| Card readability |  | Name, cost, type, effect, target, affordance |
| Enemy fairness |  | Intent shown, damage fair, status effects readable |
| Reward interest |  | Player wants at least one reward choice |
| Balance |  | No automatic win/lose strategy, boss feels harder |
| Visual identity |  | Produce fantasy + Blight tone reads immediately |
| Asset reliability |  | No broken images, consistent style, readable sizes |
| UX/accessibility |  | Keyboard, focus, contrast, labels, responsive layout |
| Performance |  | No major stutter, build size acceptable for MVP |
| Scope control |  | No distracting systems before core works |

## Pass/Fail Rule

The next build pass should focus on the lowest scoring areas.

Do not expand content until these are at least 3:

- Core loop works
- Combat clarity
- Card readability
- Enemy fairness
- Asset reliability
